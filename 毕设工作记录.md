## 第一阶段：数据集构建与预处理工作（2025年11月 - 12月）

### 1.1 数据集选择与原始状态分析
**工作描述：**
*   **数据集来源：** 选定包含多视角、多天气条件下的船舶数据集（开源 iShip-1），共计 17,237 张航拍图像。
*   **原始类别分布：** 数据集本身包含 Bulk Carrier (散货船)、Cargo Ship (货船)、Passenger Ship (客船)、Fishing Vessel (渔船)、Pleasure Craft (娱乐用船) 等多细粒度分类。
*   **原始问题剖析：**
    *   **分类冗余：** 对于“小目标检测”任务，过于细化的船舶类型分类会额外增加模型的分类难度（Classification Loss），尤其是在小目标只有十几个像素、特征极其稀疏的情况下，强行要求模型识别船型往往导致召回率下降。
    *   **尺度混缩：** 数据集中的船舶尺度差异巨大，但并没有在标签层面将“小目标”单独剥离出来用于针对性评估（mAP@s）。

### 1.2 数据预处理：类别的重定义与清洗（🌟 核心工作）
**解决策略（契合开题报告 3.1.1 节）：**
本项目的核心痛点在于“小目标”。为了将模型的学习重心从“识别船型”转移到“提高船舶检出率并重点关注微小船舶”，我实施了**类别重构与合并操作**。

**具体操作：**
1.  **定义“小目标”标准：** 放弃了 COCO 数据集基于绝对像素面积（$32\times32$）的划分标准，结合遥感图像分辨率变化极大的特性，采用**相对面积比例（Relative Area Ratio）**作为划分依据。最终设定：**目标面积占整幅图像面积的比例小于 0.5%（即 $Width_{norm} \times Height_{norm} < 0.005$）定义为小目标。**
2.  **编写自动化脚本 (`convert_labels_to_2class.py`)：** 解析了原始的 17,237 个 YOLO 格式的 `.txt` 标签文件。脚本计算每个 bbox 的归一化面积，并应用上述 $0.5\%$ 阈值。
3.  **类别映射合并规则：**
    *   不再区分原本的五类船型。
    *   **类别 `0`（常规船舶 / Normal Ship）：** 相对面积 $\ge 0.5\%$ 的目标。
    *   **类别 `1`（小目标船舶 / Small Target Ship）：** 相对面积 $< 0.5\%$ 的目标。

**最终清洗结果统计：**
通过脚本处理后，生成了全新的标签文件夹 `labels_2class`。
*   总计处理图像文件：17,237 张。
*   总计识别出边界框：33,265 个。
*   大/中型 常规船舶数（类别 `0`）：21,998 个（占比约 66.1%）。
*   极小目标 船舶数（类别 `1`）：11,267 个（占比约 33.9%）。

**这部分工作对最终论文的价值：**
*   为第四章《实验环境搭建与数据预处理》直接提供了素材。
*   消融实验和最终评估时可以直接利用类别 `1` 计算专属的 `mAP@small` 指标。
*   解决了开题报告中提到的训练样本类别极度不平衡问题。

## 第二阶段：数据集标准化与架构搭建（已完成 2026年3月）

### 2.1 原始数据清洗与语义化重命名
**工作描述：**
为了后续能精准实施“针对小目标的特异性数据增强”（如从小船图片中抠图并Copy-Paste），对原始数据集进行了语义级的重构：
1. **语义分类与重命名：** 编写 `rename_dataset.py` 脚本，读取标签并按内含船舶类型对所有图片及配套的 `txt`、`json` 文件联动原地重命名：
   - 仅包含常规船：命名为 `Normal_Ship_xxxx`
   - 仅包含极小目标：命名为 `Small_Target_Ship_xxxx` 
   - 同时包含大小目标：命名为 `Mixed_Ship_xxxx`
2. **剔除冗余：** 找出了仅有的 6 张完全无目标的 `Background_xxxx` 纯背景图，并在脚本层面实现了物理剔除，保证数据入模的纯净度。

### 2.2 项目工程化与全局路径配置 (Global Config)
**工作描述：**
为了保证消融实验中不同类别脚本和数据集组合不会发生“路径混乱与报错”，引入了全局配置机制：
1. **构建中枢 `config.py`**：利用 `pathlib` 动态统筹管理 `dataset_original` (基础矿坑)、`dataset_baseline` (基线使用集)、以及各类常量（如 0.5% 的面积阈值、IMG_SIZE 等）。
2. **规范文件结构：**
   - 建立 `dataset/` 目录，隔离所有数据处理、划分及增强脚本。
   - 建立 `train_scripts/` 目录，用于隔离不同消融维度下的独立训练启动脚本（如 `train_baseline.py`）。

### 2.3 基线数据集划分 (Train/Val/Test Split)
**工作描述：**
利用更新与重命名后的原始数据集，执行随机打乱划分：
- **比例标准：** 遵照深度学习严苛要求，按 **7:2:1** 进行 Train (70%) / Val (20%) / Test (10%) 的拆分。
- **目标产出：** 将拆分后的数据隔离输出至 `dataset_baseline` 文件夹中，并自动封装对应的 YOLO 配置描述文件 `ship_dataset.yaml`，至此基准测试（Baseline）所需的靶场已搭建完毕。

### 2.4 数据池池化设计与多维对抗数据增强 (Data Augmentation) (已完成 2026年3月)
**工作描述（契合开题报告 4.3 节）：**
为了缓解“类别 1(小目标) : 类别 0(常规目标) = 1 : 2”的严重不平衡，并大幅提升模型抗复杂环境干扰的能力，我设计了“底层迭代池”模式，即所有增强数据全部汇总至 `dataset_augmentation` 终极大池，目前已完成以下增强闭环：

1.  **特征融合：智能 Copy-Paste 提取**
    *   **实现：** 编写 `copypaste_data.py`。专门针对含有 `Small_Target_Ship` 的图像，由 OpenCV 提取目标并基于 IoU 重叠检测机制，粘贴至常规背景或空旷海面上。
    *   **成效：** 物理级补充了纯净小船目标的数量，打乱了原有的锚框分布习惯，强制模型学习“船”的特征，而非过拟合背景。

2.  **生成式对抗增强：AIGC 图生图 (Img2Img)**
    *   **实现：** 编写 `generate_aigc_img2img.py`。针对平视图中容易漏检的极限地平线远洋小船、逆光高亮（Sun Glare）以及复杂海浪白沫遮挡等问题，接通了包含 Qwen-Image-Edit 模型的 API 进行对抗性环境重绘。
    *   **规范化：** 联合编写 `integrate_aigc_data.py`，实现对手动标注的（LabelMe格式）增强图像的自动化剥离。脚本根据 0.5% 面积规则自动转为 2 类别 YOLO txt标签，将这批带有高度对抗性质的高质量坏样本统一冠名（如 `aigc_ship_000001`）并合入终极增强池。

3.  **待办：全局环境降级 (Weather Simulation)**
    *   后续将利用 `albumentations` 库，在已融合的数据集上实施全局的浓雾、雨雪遮罩，完成整个小目标数据增强链路的最后一块拼图。
    *   *注：经过对数据集视角物理特性的深度分析，决定**废弃原始计划中的 SAHI（大图切片）技术**。原因：本数据集为平视图（Eye-level），小目标呈高度集中的“同水平线管状分布”（即绝大部分集中在海平线上）。若盲目使用网格化切片，将切出海量仅含“纯天空”或“纯近海”的无效背景图，引发极其严重的正负样本失衡，干扰网络对目标的正常回归。这体现了本项目在算法设计上对实际业务场景物理规律的针对性考量。*

## 第三阶段：消融实验构架设计与基线评估

### 3.1 确立分离式“双维度”消融实验架构
与开题报告相契合，制定了最严谨的“变量控制”双表验证体系，杜绝模型改进与数据增强间的因果耦合干扰：
1. **第一维度（数据层面消融）：** 在 Baseline（原始YOLOv12）上，分别跑不同的数据增强包（如Copy-Paste、天气模拟等），论证各种增强策略的提升效益，最终整合成一个**“最优增强数据集(D-Best)”**。
2. **第二维度（模型层面消融）：** 废弃初始数据，**统一以“D-Best 最优增强数据集”为起跑线底座**，验证 P2微尺度高分辨率检测头、SPD-Conv无损下采样机制、以及EIoU损失函数这三项网络架构改动，层层叠加突破的 AP 极限。
